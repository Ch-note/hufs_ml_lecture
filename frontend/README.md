# AI English Tutor - 배포 및 공유 가이드

## Q: 다른 사람들이 이 프로토타입을 볼 수 있게 하려면 배포를 해야 하나요?

**네, 맞습니다.** 다른 사람들이 내 컴퓨터에 접속하지 않고도 스마트폰이나 PC에서 이 앱을 사용해 보려면, 인터넷상에 코드를 올려두고 접속할 수 있는 주소(URL)를 만드는 **'배포(Deployment)'** 과정을 거쳐야 합니다.

현재 만들어진 앱은 프론트엔드(React)만으로 완벽하게 동작하도록 설계되었기 때문에, 서버 구축 없이 **무료 호스팅 서비스**를 이용해 5분 만에 배포할 수 있습니다.

---

## 방법 1: Vercel을 이용한 정식 배포 (가장 추천)

가장 대중적이고 쉬운 프론트엔드 무료 배포 플랫폼인 **Vercel**을 사용하는 방법입니다.

### 1단계: GitHub에 코드 올리기
1. GitHub(github.com)에 로그인하고 새로운 Repository(저장소)를 만듭니다. (Private으로 설정하는 것을 권장합니다.)
2. 현재 작업 중인 폴더의 코드들을 해당 GitHub 저장소에 Push(업로드) 합니다.

### 2단계: Vercel에 연결하여 배포하기
1. [Vercel(vercel.com)](https://vercel.com)에 접속하여 GitHub 계정으로 로그인합니다.
2. 우측 상단의 **[Add New...] -> [Project]**를 클릭합니다.
3. 방금 코드를 올린 GitHub 저장소를 찾아 **[Import]** 버튼을 누릅니다.
4. **Configure Project** 화면에서 다음을 확인합니다:
   - Framework Preset이 `Vite`로 되어 있는지 확인합니다.
   - **Environment Variables (환경 변수)** 섹션을 펼칩니다.
   - Name에 `API_KEY`를 입력하고, Value에 본인의 **Google Gemini API 키**를 입력한 뒤 [Add]를 누릅니다.
5. **[Deploy]** 버튼을 누릅니다.
6. 1~2분 뒤 배포가 완료되면 `https://ai-english-tutor-xxx.vercel.app` 같은 접속 가능한 URL이 발급됩니다. 이 링크를 사람들에게 공유하면 됩니다!

## 🚨 [매우 중요] 보안 주의사항: API 키 노출 위험

현재 프로토타입은 빠른 개발과 지연 시간(Latency) 최소화를 위해 **프론트엔드(브라우저)에서 직접 Gemini API를 호출**하는 구조입니다. 

이 방식으로 Vercel 등에 배포할 경우, **웹사이트에 접속한 누구나 개발자 도구(F12)를 열어 당신의 Gemini API 키를 훔쳐볼 수 있습니다.** 누군가 악의적으로 API 키를 가져가 과도하게 사용하면 요금 폭탄을 맞을 수 있습니다.

### 프로토타입 단계에서의 현실적인 대처법:
1. **신뢰할 수 있는 사람에게만 링크 공유:** 팀원이나 평가자 등 소수에게만 링크를 공유하세요.
2. **테스트 종료 후 API 키 삭제:** 테스트나 시연이 끝난 직후, Google AI Studio에 접속하여 **해당 API 키를 즉시 삭제(Revoke)하거나 재생성**하세요.

*(참고: 실제 상용 서비스(Production)를 만들 때는 반드시 별도의 백엔드 서버(Node.js, Python 등)를 구축하여 API 키를 서버 쪽에 숨기고, 프론트엔드는 백엔드와 통신하는 구조로 변경해야 합니다.)*

## 방법 2: 배포 없이 내 컴퓨터를 임시로 열어주기 (Ngrok 사용)

API 키 유출이 걱정되거나, GitHub에 올리고 배포하는 과정이 번거롭다면 **Ngrok**이라는 도구를 사용해 내 컴퓨터에서 실행 중인 화면을 임시 URL로 만들어 공유할 수 있습니다.

1. 로컬에서 앱을 실행합니다. (`npm run dev` -> 보통 `http://localhost:3000`에서 실행됨)
2. [Ngrok(ngrok.com)](https://ngrok.com/) 프로그램을 다운로드하고 설치합니다.
3. 터미널을 열고 다음 명령어를 입력합니다:
   ```bash
   ngrok http 3000
   ```
4. 화면에 `Forwarding https://xxxx-xx-xx.ngrok-free.app -> http://localhost:3000` 형태의 주소가 나타납니다.
5. 이 `https://...ngrok-free.app` 주소를 사람들에게 공유하세요.
6. **장점:** API 키가 인터넷에 영구적으로 노출되지 않으며, 내 컴퓨터를 끄거나 Ngrok을 종료하면 링크도 즉시 접속 불가 상태가 되어 안전합니다. 단기 시연용으로 최고입니다.
